5 skills

Core features

다섯 개의 플러그인 스킬로 필요한 작업만 로드합니다.

배경, 투명 PNG, 벡터, 애니메이션, CSV 업로드는 실패 지점이 다릅니다. 플러그인 방식은 각 route를 좁게 나누고, 실제 제출 표면에 맞는 규칙만 불러옵니다.

JPG

jpg-background

수면, 종이 질감, 빛 반사처럼 full-bleed 배경에 맞는 source를 보존하고 Background CSV로 보냅니다.

  • source PNG 보존
  • 최종 JPG와 Background CSV 생성
  • 확장자 없는 fileName 검증
PNG

png-element

개별 source 생성, edge-connected 배경제거, Photopea 마무리로 alpha PNG 요소 품질을 지킵니다.

  • 1개씩 개별 source 생성
  • 피사체 내부 색 보존
  • 계단현상과 anti-aliasing 확대 검수
SVG

svg-beta

색상 변경 가능한 단순 벡터 요소는 5색 이하, 실제 SVG, 배경 제거 기준으로 검수합니다.

  • vector source와 viewBox 확인
  • embedded raster, script, 외부 링크 차단
  • SVG element CSV 생성
GIF

gif-beta

움직이는 일러스트 요소는 영상과 분리하고, loop playback과 투명 배경 안정성을 확인합니다.

  • frame/animation source 보존
  • 700-1920px, 25MB 이하 검증
  • GIF contentType CSV 생성

Workflow

설치, 라우팅, CSV 왕복을 분리합니다.

플러그인은 설치와 업데이트를 맡고, 각 스킬은 파일 생성과 검증만 좁게 처리합니다. DesignHub 업로드 후에는 내려받은 CSV를 기준으로 uniqueId를 보존합니다.

01 / INSTALL

Register

GitHub installer가 plugin root를 준비하고 Codex marketplace에 local source로 등록합니다.

02 / ROUTE

Load

요청에 맞춰 png-element, jpg-background, svg-beta, gif-beta, upload-csv만 선택합니다.

03 / ROUNDTRIP

Merge

업로드 후 DesignHub CSV를 다운로드하고 전체 행과 uniqueId를 보존해 metadata를 병합합니다.

플러그인 방식으로 남기는 제출 워크플로우

설치는 플러그인이 책임지고, 산출물 처리는 route별 스킬이 책임집니다. 외부 서비스 상태를 바꾸는 파일 업로드와 CSV 업로드는 Computer Use로 확인합니다.

  1. npx 또는 수동 clone으로 설치합니다.

    git checkout 기반 설치를 사용하면 Codex 새 세션에서 안전한 자동 업데이트를 받을 수 있습니다.

  2. 요청에 맞는 스킬만 로드합니다.

    PNG, JPG, SVG, GIF, CSV 업로드 route가 분리되어 과한 문맥 없이 정확한 규칙을 적용합니다.

  3. 원본과 프롬프트를 먼저 보존합니다.

    imagegen 결과물, 생성 프롬프트, batch 경로를 분리해 같은 주제를 다시 만들 수 있게 둡니다.

  4. 투명 PNG는 개별 source 기준으로 처리합니다.

    combined sheet crop을 피하고, 배경제거 후 내부 디테일과 anti-aliasing을 확대 검수합니다.

  5. DesignHub CSV는 다운로드본을 기준으로 병합합니다.

    fileName과 uniqueId는 DesignHub가 내려준 전체 CSV를 source of truth로 삼습니다.

  6. 외부 업로드 상태를 분리해 보고합니다.

    파일 업로드, CSV 업로드, 최종 심사 제출은 같은 완료 상태가 아니므로 각각 따로 확인합니다.

  7. 제출 전 눈검수와 숫자 검수를 같이 봅니다.

    흰색, 어두운 배경, 체커보드 위에서 깨짐을 확인하고 산출 폴더와 CSV가 같은 batch인지 맞춥니다.

Technical specs

플러그인 설치와 제출 전 검증 기준을 작게 남겼습니다.

의존성

  • Node 18 이상과 git
  • Codex 또는 compatible agent runtime
  • Python 3.10 이상과 Pillow
  • PNG 마무리용 Chromium 브라우저와 Photopea
  • 실제 DesignHub UI 작업용 Computer Use

출력 형식

  • jpg-background route
  • png-element route
  • svg-beta route
  • gif-beta route
  • upload-csv route

검증 기준

  • 파일 업로드 후 DesignHub CSV 다운로드
  • uniqueId 보존 병합
  • 키워드는 20-25개 사이에서 중복 제거
  • PNG 요소는 alpha와 edge 품질 확대 확인
  • 최종 심사 제출은 별도 사용자 확인 필요

라이선스

  • MIT 라이선스
  • macOS, Linux, Windows PowerShell 예시 포함
  • 한국어와 영어 스킬 문서 유지
  • GitHub Pages 공개 랜딩 페이지 지원
  • Git checkout 설치본 자동 업데이트 hook 포함
