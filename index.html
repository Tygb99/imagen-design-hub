imagegen으로 만든 배경 JPG, 투명 PNG 요소, DesignHub CSV를 같은 규칙으로 정리하는 Codex skill입니다. source 보존부터 파일명, DPI, contentType, 키워드 검증까지 업로드 직전 표면을 한 번에 맞춥니다.

Route selection

배경은 JPG로, 요소는 PNG로. 헷갈리는 분기를 스킬이 먼저 잡습니다.

DesignHub에서 거절되기 쉬운 지점은 대개 생성 품질이 아니라 파일 표면입니다. 이 페이지는 스킬이 고정하는 산출물 경로와 검증 기준을 보여줍니다.

Imagegen background route

수면, 종이 질감, 패턴, 빛반사 같은 full-bleed 배경은 source PNG를 보존하고 최종 JPG로 변환합니다.

contentType=Background

Transparent element route

투명 요소는 단색 크로마키 생성, helper alpha, Photopea crop/DPI 마무리 순서로 처리합니다.

contentType=PNG element
CSV

Metadata repair route

확장자가 붙은 fileName, 임의 uniqueId, 제작 과정 키워드를 업로드 전에 제거합니다.

Install and run

Clone 
git clone https://github.com/Tygb99/imagen-design-hub.git
macOS / Linux 
python -m pip install -r requirements.txt
python scripts/remove_chroma_key.py \
  --input source.png \
  --out final-alpha.png \
  --auto-key border \
  --soft-matte \
  --despill
Windows PowerShell 
py -3 -m pip install -r requirements.txt
py -3 ./scripts/remove_chroma_key.py `
  --input "./source.png" `
  --out "./final-alpha.png" `
  --auto-key border `
  --soft-matte `
  --despill

Validation surface

랜딩 페이지는 스킬 소개만이 아니라 유지보수 기준입니다. 어떤 포맷을 만들고 무엇을 검증해야 하는지 바로 보이게 했습니다.

Source preserved

생성 원본은 workspace 안에 복사하고, 변환본과 리뷰 시트를 별도 폴더로 둡니다.

Filename clean

DesignHub CSV의 fileName은 확장자 없이 최종 파일 stem과 매칭됩니다.

UniqueId safe

파일 업로드 전 CSV에서는 uniqueId를 비워 두고, DesignHub 제공 값만 사용합니다.

Photopea bounded

Photopea는 배경 제거 대체가 아니라 PNG 요소의 crop, resize, DPI 마무리에만 씁니다.
