imagegen으로 만든 배경 JPG와 투명 PNG 요소를 DesignHub 제출 표면까지 정리합니다. 원본 보존, 포맷 변환, CSV 검증, Photopea 마무리를 한 흐름으로 묶었습니다.

Core features

프리미엄 디자인 자산 생성부터 제출 전 검수까지 자동화합니다.

배경과 요소는 요구 포맷이 다릅니다. 이 스킬은 생성 품질보다 먼저 깨지는 파일명, uniqueId, contentType, DPI, 키워드 기준을 한 화면에서 정리합니다.

JPG

JPG 배경 생성

수면, 종이 질감, 빛 반사처럼 full-bleed 배경에 맞는 source를 보존하고 최종 JPG와 CSV를 분리합니다.

  • source PNG 보존
  • 최종 JPG와 Background CSV 생성
  • 확장자 없는 fileName 검증
PNG

투명 PNG 처리

크로마키 제거와 Photopea 마무리 흐름을 분리해, alpha PNG 요소가 업로드 전 표면을 잃지 않게 합니다.

  • 자동 크로마키 제거
  • Photopea crop, resize, DPI 마무리
  • PNG element CSV 생성

Workflow

세 단계로 끝나는 간단하고 강력한 프로세스

생성, 처리, 제출 준비를 분리해서 실패 원인을 빠르게 찾고, 같은 배치를 다시 만들 수 있게 합니다.

01 / PROMPT

Generate

imagegen source를 생성하고 프롬프트, 원본, batch 경로를 분리해 보존합니다.

02 / PROCESS

Refine

배경은 JPG로 변환하고, 요소는 chroma helper와 Photopea로 alpha 표면을 정리합니다.

03 / DELIVER

Validate

DesignHub CSV의 fileName, uniqueId, contentType, 키워드 수를 업로드 전에 검증합니다.

텍스트로 남기는 제출 워크플로우

이미지는 보기 좋게, 설명은 검수자가 따라갈 수 있게 분리합니다. 아래 순서를 기준으로 배경 JPG와 PNG 요소를 나누고 CSV 오류를 업로드 전에 줄입니다.

  1. 원본과 프롬프트를 먼저 보존합니다.

    imagegen 결과물, 생성 프롬프트, batch 경로를 분리해 같은 주제를 다시 만들 수 있게 둡니다.

  2. 배경은 JPG 전용 흐름으로 보냅니다.

    수면, 물빛, 패턴처럼 화면을 채우는 이미지는 JPG로 저장하고 배경용 metadata만 연결합니다.

  3. 요소는 PNG 전용 흐름으로 보냅니다.

    피사체가 있는 이미지는 chroma cutout과 Photopea 마무리로 투명도, crop, DPI를 확인합니다.

  4. CSV는 DesignHub 규칙에 맞춥니다.

    fileName에는 확장자를 빼고, uniqueId는 업로드 후 받은 값을 유지하며, 키워드는 20~25개로 정리합니다.

  5. 제출 전 눈검수와 숫자 검수를 같이 봅니다.

    흰색, 어두운 배경, 체커보드 위에서 깨짐을 확인하고 산출 폴더와 CSV가 같은 batch인지 맞춥니다.

Technical specs

제출 전에 확인할 기준을 작게, 또렷하게 남겼습니다.

의존성

  • Python 3.10 이상
  • Pillow 이미지 처리 라이브러리
  • NumPy 크로마키 처리
  • Chromium 기반 브라우저와 Photopea

출력 형식

  • 고해상도 JPG 배경 이미지
  • 투명 alpha PNG 요소
  • DesignHub CSV 메타데이터
  • 프롬프트 로그와 리뷰 시트

검증 기준

  • fileName은 확장자 없이 stem만 사용
  • 업로드 전 uniqueId는 빈칸 유지
  • 키워드는 20-25개 사이에서 중복 제거
  • PNG 요소는 WebP 없이 alpha PNG로 유지

라이선스

  • MIT 라이선스
  • macOS, Linux, Windows PowerShell 예시 포함
  • 한국어와 영어 스킬 문서 유지
  • GitHub Pages 공개 랜딩 페이지 지원
